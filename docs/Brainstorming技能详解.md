# Brainstorming 技能详解

## 功能概述

Brainstorming（头脑风暴）技能用于**在任何创造性工作之前**探索用户意图、需求和设计方案。它通过结构化的对话引导你：

1. **深入理解需求** - 不只是表面要求，而是背后的真正目标
2. **识别潜在问题** - 发现可能的陷阱和边界情况
3. **生成多种方案** - 提供不同的实现路径供选择
4. **明确验收标准** - 确定什么才算完成

## 何时使用

在以下情况**必须**使用 brainstorming：

- 创建新功能或组件
- 添加新的功能模块
- 修改现有行为
- 任何需要做出设计决策的任务
- 用户说"我想做X"或者"帮我实现Y"

## 使用方法

### 方法 1：直接调用技能
```
/superpowers:brainstorming
```

### 方法 2：在对话中使用
当你作为 Claude 在帮助用户时，应该：
```
functions.Skill:0 {"skill": "superpowers:brainstorming"}
```

## Brainstorming 的工作流程

### 1. 启动阶段
- 技能会分析当前上下文
- 提出探索性问题
- 建立对话框架

### 2. 探索阶段（会问的问题示例）
- "这个功能解决什么核心问题？"
- "谁将使用这个功能？"
- "有什么技术限制或偏好？"
- "预计的数据量/用户量是多少？"
- "什么情况下这个功能会失败？"
- "优先级是什么？（性能/可维护性/开发速度）"

### 3. 方案生成阶段
- 提供 2-4 个不同的实现方案
- 每个方案的优缺点
- 技术选型建议
- 风险评估

### 4. 决策阶段
- 让用户选择方案（或使用 AskUserQuestion 工具）
- 明确技术栈和架构
- 确定验收标准

### 5. 后续工作
- 生成完成后，通常会引导到**writing-plans** 技能
- 编写详细的实现计划
- 然后进入执行阶段

## 实际示例

### 用户请求：
"帮我添加用户认证系统"

### 错误做法：
直接开始写代码实现 JWT 认证

### 正确做法：
1. 调用 brainstorming 技能
2. 讨论：
   - 使用场景（Web/移动端/API）
   - 用户量级
   - 现有技术栈
   - 是否需要第三方登录
   - 安全要求级别
   - 是否需要刷新 token
3. 生成方案：
   - 方案 A：简单的 JWT + 密码
   - 方案 B：完整的 OAuth2 流程
   - 方案 C：使用 Auth0/Firebase Auth
4. 用户选择后
5. 调用 writing-plans 编写详细计划
6. 执行计划

## Brainstorming 的输出

- 用户需求分析总结
- 技术选型决策
- 架构设计要点
- 验收检查清单
- 潜在风险点

## 为什么必须使用

根据 superpowers 原则：

- 提前发现问题，避免返工
- 确保理解用户真实需求
- 生成最优方案而非第一反应
- 明确完成标准
